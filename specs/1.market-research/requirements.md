# market-research — 需求规格

## 概述

聚合板块轮动与宏观消息面数据，通过 AI 解读，为用户提供"打开即可看"的每日行情研判面板。

## 项目信息

- 项目名: trade-pilot
- 架构类型: pnpm monorepo (Turborepo) — React + Vite 前端 / Hono + tRPC 后端 / Drizzle + PostgreSQL

## 需求版本

| 日期       | 版本 | 说明     |
| ---------- | ---- | -------- |
| 2026-06-16 | v1   | 初始需求 |

## 用户故事

- 作为个人交易者，我想要打开 APP 即看到当前最热门的板块及龙头币种，以便快速判断市场热点在哪里。
- 作为个人交易者，我想要看到带 AI 一句话解读的宏观要闻，以便了解可能影响加密市场的重大事件，而不是阅读大量原始新闻。

## 功能需求

1. [F-001] **板块轮动**：从 CoinGecko 获取当前热门板块（Top 10），每个板块展示热度评分、代表性代币列表（Top 3–5）及各代币涨跌幅。
2. [F-002] **龙头识别**：在每个板块内，按 24h 涨幅 + 成交量综合排序，展示板块龙头代币。
3. [F-003] **AI 板块研判**：调用用户配置的 LLM，对 Top 3 热门板块生成一段综合研判摘要（为什么热、持续性判断）。
4. [F-004] **宏观要闻流**：从 CoinGecko 新闻 API 拉取近 24h 内的加密相关要闻（Top 20 条）。
5. [F-005] **AI 新闻解读**：每条要闻附带 AI 生成的"一句话影响解读"（为什么重要、可能影响哪个方向）。
6. [F-006] **定时刷新**：板块数据每 5 分钟后端刷新缓存；新闻每 15 分钟刷新；前端展示上次更新时间。
7. [F-007] **无需配置**：无需用户手动设置数据源，打开页面即可查看（LLM API Key 配置后 AI 功能自动启用，未配置则跳过 AI 解读）。

## 非功能需求

- **性能**：页面初次加载 < 2s（数据来自后端缓存，不在请求链路上实时调 CoinGecko）。
- **可用性**：CoinGecko API 失败时展示上次缓存数据 + 提示"数据可能过时"。
- **安全**：CoinGecko API Key（如有）存储在服务端环境变量，不暴露给前端。

## 验收标准

- [ ] [AC-001] 用户打开行情研判页，能看到至少 5 个热门板块及其代表代币。
- [ ] [AC-002] 每个板块展示代币的 24h 涨跌幅。
- [ ] [AC-003] 配置 LLM 后，Top 3 板块显示 AI 研判摘要。
- [ ] [AC-004] 宏观要闻列表展示至少 10 条近期新闻。
- [ ] [AC-005] 配置 LLM 后，每条新闻旁显示 AI 一句话解读。
- [ ] [AC-006] 页面显示"上次更新时间"。
- [ ] [AC-007] 未配置 LLM 时，页面正常展示行情数据，AI 区域显示"请先配置 LLM API Key"提示。

## 依赖

- CoinGecko API（免费 tier，Public API）
- 用户配置的 LLM（OpenAI / Anthropic，见 Feature 设置模块）
- `packages/llm`（LLM 抽象层，本 feature 内新建）

## 开放问题

- CoinGecko 免费 tier 每分钟 10–30 次限制，需在后端做好缓存和限速控制。
- AI 解读调用频率：板块研判是批量一次调用还是按需生成？MVP 建议批量预生成并缓存。
