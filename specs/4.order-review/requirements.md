# order-review — 需求规格

## 概述

从 Binance 同步已平仓订单，对每笔交易自动生成 AI 复盘报告；用户可选填开/平仓逻辑，获得更精准的"对照式诊断"，将每笔交易变为可学习的经验。

## 项目信息

- 项目名: trade-pilot
- 架构类型: pnpm monorepo (Turborepo) — React + Vite 前端 / Hono + tRPC 后端 / Drizzle + PostgreSQL

| 日期       | 版本 | 说明     |

| 2026-06-16 | v1   | 初始需求 |

## 用户故事

- 作为个人交易者，我想要自动拉取我在 Binance 的已平仓订单，以便不需要手动导出数据就能开始复盘。
- 作为个人交易者，我想要 AI 对每笔交易自动生成盈亏分析，即便我没有填写任何理由。
- 作为个人交易者，我想要填写我当时开/平仓的理由，让 AI 结合我的逻辑给出诊断，以便找到真正的改进点。

## 功能需求

1. [F-001] **订单同步**：后端定时（每 15 分钟）从 Binance 拉取用户的历史成交记录（`GET /api/v3/myTrades`），写入 `trade_orders` 表（已由 Feature 3 定义，本 feature 补充同步字段）。跳过已同步记录（按 Binance orderId 去重）。
2. [F-002] **订单结构化**：对每条已平仓订单（买卖成对匹配），计算：持仓时长、盈亏金额（PnL）、盈亏百分比、入场/出场价格。
3. [F-003] **AI 基础复盘**：对每笔完整交易（有对应平仓记录），调用 LLM 自动生成复盘报告（技术面判断、时机好坏、改进建议）。
4. [F-004] **用户逻辑输入**：用户可为订单填写"开仓理由"和"平仓理由"（文本输入）。
5. [F-005] **AI 对照诊断**：填写逻辑后，AI 结合用户的主观逻辑 vs 实际结果做对照诊断（做对了什么、做错了什么、具体改进点）。
6. [F-006] **订单列表**：展示所有已同步订单，按时间倒序，支持按交易对筛选和盈亏状态筛选（盈利/亏损/未平仓）。
7. [F-007] **订单详情**：点击订单展示详情页，含基础数据 + AI 复盘报告 + 用户逻辑输入区域 + AI 对照诊断（如已填写逻辑）。
8. [F-008] **按需拉取**：提供"立即同步"按钮，触发后端立即拉取最新订单（不等定时任务）。

## 非功能需求

- **数据安全**：拉取订单使用用户已绑定的 API Key（只读权限即可，Feature 3 已存储），不需要交易权限。
- **AI 复盘生成时机**：订单同步后异步生成（不阻塞订单列表展示），生成完成后展示 badge 提示"复盘已生成"。
- **可读性**：AI 复盘报告语气克制、具体，避免空泛点评；给出 1–3 个可执行的具体改进点。

## 验收标准

- [ ] [AC-001] 绑定 API Key 后，订单列表展示从 Binance 同步的历史成交订单。
- [ ] [AC-002] 每条订单显示：交易对、方向、成交价、盈亏额、盈亏%、持仓时长。
- [ ] [AC-003] 有 AI 复盘的订单显示"AI 复盘"badge，点击详情可查看完整报告。
- [ ] [AC-004] 用户填写开/平仓理由后，AI 生成对照诊断，展示在详情页中。
- [ ] [AC-005] 按交易对筛选功能正常工作。
- [ ] [AC-006] "立即同步"按钮触发后，刷新订单列表（含 loading 状态）。
- [ ] [AC-007] 未绑定 API Key 时，订单列表展示"请先绑定 Binance API Key"。

## 依赖

- Feature 3（semi-auto-trading）中的 `trade_orders` 表和 Binance API Key 读取服务
- `packages/llm`（Feature 1 中已建立）
- Binance REST API（`GET /api/v3/myTrades`，只读权限）

## 开放问题

- 买卖对匹配（开仓 + 平仓配对）逻辑：MVP 按 symbol + 时间顺序简单配对（先买后卖），不做复杂的 FIFO 成本计算。
- 历史订单拉取深度：Binance `/api/v3/myTrades` 最多返回近 500 条，MVP 只拉取近 90 天，初次绑定时全量同步一次。
