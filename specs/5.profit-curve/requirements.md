# profit-curve — 需求规格

## 概述

从账户整体视角分析资金曲线（盈利曲线），在连续盈亏等情绪化节点主动介入 AI 冷静复盘，充当"冷静的第三方"，帮助用户避免上头或恐慌。

## 项目信息

- 项目名: trade-pilot
- 架构类型: pnpm monorepo (Turborepo) — React + Vite 前端 / Hono + tRPC 后端 / Drizzle + PostgreSQL

## 需求版本

| 日期       | 版本 | 说明     |
| ---------- | ---- | -------- |
| 2026-06-16 | v1   | 初始需求 |

## 用户故事

- 作为个人交易者，我想要看到我的账户资金曲线随时间的变化，以便直观了解自己的整体交易表现，而不只是盯着单笔盈亏。
- 作为个人交易者，我希望在连续亏损或连续盈利时，APP 能主动提醒我并给出一个 AI 冷静复盘，以便避免情绪化决策。
- 作为个人交易者，我想要定期收到账户风险提示（如最大回撤超过阈值），以便及时审视风险暴露。

## 功能需求

1. [F-001] **资金快照采集**：每日一次从 Binance 账户接口拉取当前 USDT 余额（总权益），写入 `capital_snapshots` 表，形成时间序列。
2. [F-002] **资金曲线展示**：展示资金曲线折线图，支持时间范围筛选（7天 / 30天 / 90天 / 全部），X 轴为日期，Y 轴为账户权益（USDT）。
3. [F-003] **关键指标计算**：在曲线图下方展示统计指标：
   - 最大回撤（Max Drawdown）及其发生时间段
   - 盈利天数 / 亏损天数占比
   - 近期连续盈利次数 / 连续亏损次数（基于已平仓订单，非日期）
   - 总体盈亏额（首笔交易至今）
4. [F-004] **连续盈亏触发检测**：每次订单同步完成后，检测用户当前连续盈利次数是否 ≥ N（默认 3），或连续亏损次数是否 ≥ N（默认 3），触发 AI 介入复盘生成。
5. [F-005] **AI 阶段性复盘（触发式）**：当触发条件满足时，调用 LLM 生成一段冷静的近期总结，内容包括：近期交易客观统计、情绪风险提示（连续亏损可能的心理影响）、1–2 条行动建议。语气克制，不做预测，不做投资建议。
6. [F-006] **定期风险提示**：每周一次（后端 cron），若最大回撤 > 20%，自动生成风险提示记录，前端以 banner 或通知形式展示。
7. [F-007] **未读提醒**：AI 阶段性复盘和风险提示有"未读"状态，展示红点 badge，用户查看后标记为已读。

## 非功能需求

- **AI 语气**：AI 复盘定位为"提醒"而非"交易建议"，必须在输出中包含风险声明。
- **数据来源**：资金快照来自 Binance 账户接口，连续盈亏统计来自本地 `trade_orders` 表（不重新请求 Binance）。
- **触发去重**：同一触发条件（如连续亏损 3 次）不重复触发，直到条件解除后重新满足才再次触发。

## 验收标准

- [ ] [AC-001] 资金曲线页展示折线图，时间范围切换正常工作。
- [ ] [AC-002] 关键指标（最大回撤、连续盈亏次数）展示正确数值。
- [ ] [AC-003] 连续亏损 3 次后，AI 阶段性复盘出现在页面提醒区域，语气克制。
- [ ] [AC-004] 连续盈利 3 次后，同样触发 AI 提醒（防止上头）。
- [ ] [AC-005] 最大回撤 > 20% 时，每周定期提示出现。
- [ ] [AC-006] 未读 AI 复盘有 badge 提示；查看后消失。
- [ ] [AC-007] 未绑定 API Key 时，曲线页展示"请先绑定 Binance API Key"引导。

## 依赖

- Feature 3（semi-auto-trading）：Binance API Key 读取
- Feature 4（order-review）：`trade_orders` 表中的 PnL 数据（连续盈亏统计基于此）
- `packages/llm`（Feature 1 中已建立）
- Binance REST API（`GET /api/v3/account`，需只读权限，返回 balances）

## 开放问题

- 连续 N 次阈值（当前 N=3）：可考虑在设置页开放用户自定义，MVP 先硬编码。
- "账户权益"定义：MVP 仅统计 USDT 现货余额（不含未平仓合约盈亏、币种持仓市值），在 UI 中标注说明。
