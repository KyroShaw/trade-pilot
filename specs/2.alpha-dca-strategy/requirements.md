# alpha-dca-strategy — 需求规格

## 概述

定时采集 Binance Alpha 项目行情数据，通过量化策略 + AI 辅助，筛选出"底部长期盘整"的标的，生成定投候选清单，作为产品的核心差异化功能。

## 项目信息

- 项目名: trade-pilot
- 架构类型: pnpm monorepo (Turborepo) — React + Vite 前端 / Hono + tRPC 后端 / Drizzle + PostgreSQL

## 需求版本

| 日期       | 版本 | 说明     |
| ---------- | ---- | -------- |
| 2026-06-16 | v1   | 初始需求 |

## 用户故事

- 作为个人交易者，我想要看到当前有哪些 Binance Alpha 项目处于底部盘整阶段，以便制定定投计划博取拉盘收益。
- 作为个人交易者，我想要了解每个候选标的入选的原因（量化指标），以便自行判断信号可靠性。

## 功能需求

1. [F-001] **Alpha 项目管理**：支持维护一份 Binance Alpha 项目列表（交易对符号 + 项目名），可通过后台配置或手动更新。MVP 阶段通过 DB 种子数据初始化，支持管理员通过 tRPC 增删。
2. [F-002] **行情数据采集**：对列表中每个项目，定时从 Binance REST API 拉取：K 线数据（日线，近 90 天）、当前价格、24h 成交量、历史最高价（ATH）。采集频率：每 6 小时一次。
3. [F-003] **盘整策略筛选**：量化判断"底部长期盘整"，入选条件（满足以下全部）：
   - 盘整时长 ≥ 14 天（价格在某范围内震荡）
   - 价格波动收敛（近 14 天 ATR 低于 20 日均值）
   - 距历史最高价回撤 ≥ 60%
   - 当前价格在近 90 天低位区间（< 30th 百分位）
4. [F-004] **AI 辅助研判**：对入选标的，调用 LLM 生成一段可解释的"入选理由摘要"（结合量化指标描述为什么这个项目值得关注）。
5. [F-005] **信号展示**：展示定投候选清单，每个标的显示：项目名、当前价格、盘整天数、回撤幅度、AI 入选理由、风险提示。
6. [F-006] **风险提示**：所有信号旁显示固定免责声明："本信号基于'Binance Alpha 项目必然拉盘'策略假设，不构成投资建议，请自行评估风险。"
7. [F-007] **历史信号记录**：每次筛选结果记录到 DB，保留信号生成时间，供后续准确率回溯（展示留后续迭代）。

## 非功能需求

- **数据时效**：行情数据每 6 小时刷新，展示上次更新时间。
- **可解释性**：每个信号必须展示具体量化指标数值（盘整天数、回撤幅度），不只是 AI 摘要。
- **安全**：Binance 公共市场数据端点（K 线、价格）无需 API Key，避免使用用户的交易 API Key 做数据采集。

## 验收标准

- [ ] [AC-001] 用户打开 Alpha 定投页能看到当前候选标的列表。
- [ ] [AC-002] 每个候选标的展示：项目名、价格、盘整天数、距 ATH 回撤幅度。
- [ ] [AC-003] 配置 LLM 后，每个候选标的展示 AI 入选理由摘要。
- [ ] [AC-004] 每个标的旁显示固定风险提示。
- [ ] [AC-005] 若当前无符合条件的候选标的，展示"暂无符合条件的 Alpha 定投标的"。
- [ ] [AC-006] 展示数据上次刷新时间。

## 依赖

- Binance 公共 REST API（K 线：`/api/v3/klines`，价格：`/api/v3/ticker/price`）
- `packages/llm`（Feature 1 中已建立）
- Binance Alpha 项目初始列表（种子数据）

## 开放问题

- Binance Alpha 项目列表需人工维护（无官方公开 API）；MVP 使用种子数据初始化，后续考虑爬取 Binance 官网 Alpha 页面或对接社区数据源。
- ATH（历史最高价）定义：取 Binance 上架以来最高，还是特定窗口期（90 天）内最高？MVP 使用上架以来所有 K 线数据的最高价。
